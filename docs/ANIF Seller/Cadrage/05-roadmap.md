# Roadmap

[← Retour](README.md)

## Phase 0 — Recherche IA et cadrage

Durée indicative : 1 à 2 semaines.

Objectif : cadrer le problème avant de développer trop loin, sans organiser d'interviews au démarrage.

Actions :

- rechercher les données publiques sur le digital, le social commerce et le mobile money ;
- benchmarker WhatsApp Business, Shopify Starter, Flutterwave Store, Bumpa et outils proches ;
- identifier les outils probables des vendeurs : WhatsApp, carnet, Excel, Instagram ;
- formuler les douleurs vendeurs comme hypothèses provisoires ;
- définir une promesse produit provisoire ;
- préparer un brief IA pour le MVP.

Livrables :

- synthèse de recherche sourcée ;
- benchmark concurrentiel ;
- liste d'hypothèses produit ;
- liste des fonctionnalités indispensables ;
- première hypothèse tarifaire ;
- brief IA du MVP.

## Phase 1 — MVP utilisable

Durée indicative : 3 à 6 semaines.

Objectif : créer la première version exploitable par des vendeurs pilotes.

Fonctionnalités :

- compte vendeur ;
- CRUD produits ;
- boutique publique ;
- création de commandes ;
- dashboard commandes ;
- changement de statut ;
- messages WhatsApp générés.

Critère de sortie :

- au moins 5 vendeurs pilotes peuvent gérer leurs produits et commandes.

## Phase 2 — Beta commerciale

Durée indicative : 1 à 2 mois.

Objectif : transformer l'outil en produit payable.

Fonctionnalités :

- statistiques simples ;
- limites d'offre gratuite ;
- page abonnement ;
- export commandes ;
- amélioration UX mobile ;
- onboarding vendeur ;
- système de support simple.

Critère de sortie :

- premiers clients payants.

## Phase 3 — Paiements et automatisation

Durée indicative : 2 à 3 mois après beta.

Objectif : augmenter la valeur produit avec les paiements locaux et l'automatisation.

Fonctionnalités :

- paiement FedaPay ou KkiaPay ;
- suivi automatique du statut paiement ;
- notifications client ;
- relances de paiement ;
- historique client enrichi.

Critère de sortie :

- les vendeurs peuvent recevoir et suivre des paiements depuis l'application.

## Phase 4 — SaaS multi-vendeurs solide

Objectif : passer d'un outil MVP à une plateforme stable.

Fonctionnalités :

- gestion des abonnements ;
- rôles équipe : propriétaire, vendeur, préparateur ;
- boutique personnalisable ;
- domaine personnalisé ;
- analytics avancées ;
- templates par secteur ;
- système de facturation.

## Phase 5 — Expansion

Objectif : élargir le marché.

Pistes :

- version anglophone ;
- intégration plus forte Instagram/Facebook ;
- application mobile ou PWA avancée ;
- réseau de livreurs partenaires ;
- marketplace optionnelle des boutiques actives ;
- offres par secteur : mode, restauration, cosmétiques, électronique.
