# Modèle économique

[← Retour](README.md)

## Objectif

Monétiser ANIF Seller avec un abonnement mensuel accessible, adapté aux petits commerces, puis ajouter des revenus optionnels sur les paiements, services et fonctionnalités premium.

## Tarification proposée

| Offre | Prix indicatif | Cible | Limites |
|-------|----------------|-------|---------|
| Gratuit | 0 FCFA/mois | Test et acquisition | 10 produits, 20 commandes/mois |
| Starter | 2 000 à 5 000 FCFA/mois | Petit vendeur actif | 100 produits, commandes illimitées simples |
| Pro | 10 000 FCFA/mois | Boutique structurée | statistiques, exports, plusieurs utilisateurs |
| Business | Sur devis | Commerce avec équipe | multi-boutiques, support prioritaire, options avancées |

## Sources de revenus

### 1. Abonnement SaaS

Le revenu principal vient d'un paiement mensuel. Le prix doit rester assez bas pour être acceptable, mais assez clair pour prouver la valeur.

### 2. Commission sur paiements

Quand FedaPay ou KkiaPay est intégré, ANIF Seller peut ajouter une petite commission de service sur les paiements réussis, si cela reste acceptable pour le vendeur.

### 3. Services d'installation

Certains vendeurs peuvent payer pour être accompagnés :

- ajout initial des produits ;
- création du visuel de boutique ;
- formation rapide ;
- configuration paiement et livraison.

### 4. Options premium

Fonctionnalités payantes possibles :

- domaine personnalisé ;
- statistiques avancées ;
- relances clients ;
- exports comptables ;
- gestion équipe ;
- messages promotionnels ;
- templates de boutique.

## Stratégie de lancement

### Phase 1 — Pilotes gratuits

Trouver 5 à 10 vendeurs proches :

- vendeurs WhatsApp ;
- commerces familiaux ;
- boutiques de vêtements ;
- vendeurs Instagram ;
- restaurateurs.

Objectif : observer leur usage réel et corriger les frictions.

### Phase 2 — Offre fondatrice

Proposer un tarif simple aux premiers utilisateurs sérieux :

- 2 000 FCFA/mois pendant 3 mois ;
- support direct ;
- migration gratuite de leurs produits ;
- feedback prioritaire.

### Phase 3 — Offre publique

Quand le produit devient stable, lancer les offres Starter et Pro avec une page de paiement claire.

## Risques business

| Risque | Réponse possible |
|--------|------------------|
| Les vendeurs ne veulent pas payer | Prouver le gain de temps avec une version gratuite limitée |
| Les vendeurs abandonnent l'outil | Rendre la saisie produit et le suivi commande très rapides |
| WhatsApp suffit déjà | Positionner l'app comme un complément à WhatsApp |
| Support utilisateur chronophage | Créer des tutoriels courts et une interface très simple |
| Paiements locaux complexes | Démarrer sans paiement automatique, puis intégrer FedaPay/KkiaPay |

## Indicateurs à suivre

- Nombre de boutiques créées
- Nombre de produits publiés
- Nombre de commandes reçues
- Nombre de vendeurs actifs chaque semaine
- Taux de conversion gratuit vers payant
- Revenu mensuel récurrent
- Motifs d'abandon
